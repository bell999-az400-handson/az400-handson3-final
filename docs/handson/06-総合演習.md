**Step 4: SSH鍵の確認と準備**

Azure Pipelines からVMにSSH接続するための鍵を確認・準備します。

```powershell
# 既存のSSH鍵を確認
ls $env:USERPROFILE\.ssh\ | Format-Table Name, LastWriteTime

# id_rsa が存在するか確認
if (Test-Path $env:USERPROFILE\.ssh\id_rsa) {
    Write-Host "✅ SSH鍵ペア (id_rsa) が既に存在します" -ForegroundColor Green
    
    # 公開鍵の内容を表示
    Write-Host "`n公開鍵の内容:" -ForegroundColor Cyan
    Get-Content $env:USERPROFILE\.ssh\id_rsa.pub
    
    # VMに登録されているか確認
    Write-Host "`nVMへのSSH接続テスト..." -ForegroundColor Cyan
    ssh azureuser@20.89.144.177 "echo '✅ SSH接続成功'"
    
    if ($LASTEXITCODE -eq 0) {
        Write-Host "✅ 既存の鍵でVMに接続できます。この鍵を使用してください。" -ForegroundColor Green
    } else {
        Write-Host "⚠️ SSH接続失敗。公開鍵をVMに追加する必要があります。" -ForegroundColor Yellow
    }
} else {
    Write-Host "⚠️ SSH鍵が見つかりません。新しく作成します..." -ForegroundColor Yellow
    
    # 新しいSSH鍵ペアを生成
    ssh-keygen -t rsa -b 4096 -f $env:USERPROFILE\.ssh\id_rsa -N '""'
    
    # 公開鍵をVMに追加
    $publicKey = Get-Content $env:USERPROFILE\.ssh\id_rsa.pub -Raw
    az vm user update `
      --resource-group rg-az400-handson3 `
      --name az400-vm `
      --username azureuser `
      --ssh-key-value $publicKey
    
    Write-Host "✅ 新しいSSH鍵を作成し、VMに登録しました" -ForegroundColor Green
}
```

> **🎯 AZ-400試験のポイント**: 
> - SSH鍵認証はパスワード認証より安全
> - 秘密鍵（`id_rsa`）は厳重に管理し、共有しない
> - 公開鍵（`id_rsa.pub`）のみをVMに配置
> - 既存の鍵がある場合は再利用可能

#### 2.4 Users Controller の作成

`backend/Controllers/UsersController.cs`:
```csharp
using Microsoft.ApplicationInsights;
using Microsoft.AspNetCore.Mvc;
using Microsoft.EntityFrameworkCore;
using backend.Data;
using backend.Models;

namespace backend.Controllers;

[ApiController]
[Route("api/[controller]")]
public class UsersController : ControllerBase
{
    private readonly UserDbContext _context;
    private readonly TelemetryClient _telemetry;
    private readonly ILogger<UsersController> _logger;

    public UsersController(UserDbContext context, TelemetryClient telemetry, ILogger<UsersController> logger)
    {
        _context = context;
        _telemetry = telemetry;
        _logger = logger;
    }

    [HttpGet]
    public async Task<ActionResult<IEnumerable<User>>> GetUsers()
    {
        _telemetry.TrackEvent("GetUsers");
        _logger.LogInformation("Getting all users");
        return await _context.Users.ToListAsync();
    }

    [HttpGet("{id}")]
    public async Task<ActionResult<User>> GetUser(int id)
    {
        _telemetry.TrackEvent("GetUser", new Dictionary<string, string> { { "UserId", id.ToString() } });
        var user = await _context.Users.FindAsync(id);

        if (user == null)
        {
            _logger.LogWarning("User {UserId} not found", id);
            return NotFound();
        }

        return user;
    }

    [HttpPost]
    public async Task<ActionResult<User>> CreateUser(User user)
    {
        _telemetry.TrackEvent("CreateUser");
        _context.Users.Add(user);
        await _context.SaveChangesAsync();
        _logger.LogInformation("User {UserId} created", user.Id);

        return CreatedAtAction(nameof(GetUser), new { id = user.Id }, user);
    }
}
```

### Exercise 3: CI/CD パイプラインの構築

#### 3.1 完全な YAML パイプライン

`azure-pipelines.yml`:
```yaml
# azure-pipelines.yml - Complete CI/CD Pipeline
trigger:
  branches:
    include:
    - main
  paths:
    include:
    - backend/**

pr:
  branches:
    include:
    - main

variables:
  buildConfiguration: 'Release'
  NUGET_PACKAGES: $(Pipeline.Workspace)/.nuget/packages
  # System.Debug: true  # デバッグ時のみ有効化

stages:
# ステージ 1: ビルドとテスト
- stage: Build
  displayName: 'Build and Test'
  jobs:
  - job: BuildJob
    displayName: 'Build API'
    pool:
      vmImage: 'ubuntu-latest'
    steps:
    # .NET のセットアップ
    - task: UseDotNet@2
      displayName: 'Use .NET 8.0'
      inputs:
        version: '8.0.x'
    
    # NuGet キャッシュ
    - task: Cache@2
      displayName: 'Cache NuGet Packages'
      inputs:
        key: 'nuget | "$(Agent.OS)" | **/packages.lock.json'
        restoreKeys: |
          nuget | "$(Agent.OS)"
        path: $(NUGET_PACKAGES)
    
    # 依存関係の復元
    - task: DotNetCoreCLI@2
      displayName: 'Restore Dependencies'
      inputs:
        command: 'restore'
        projects: 'backend/**/*.csproj'
    
    # ビルド
    - task: DotNetCoreCLI@2
      displayName: 'Build Solution'
      inputs:
        command: 'build'
        projects: 'backend/**/*.csproj'
        arguments: '--configuration $(buildConfiguration) --no-restore'
    
    # テスト（テストプロジェクトがある場合）
    - task: DotNetCoreCLI@2
      displayName: 'Run Tests'
      inputs:
        command: 'test'
        projects: 'backend/**/*Tests.csproj'
        arguments: '--configuration $(buildConfiguration) --no-build --collect:"XPlat Code Coverage"'
    
    # コードカバレッジレポート
    - task: PublishCodeCoverageResults@1
      displayName: 'Publish Code Coverage'
      inputs:
        codeCoverageTool: 'Cobertura'
        summaryFileLocation: '$(Agent.TempDirectory)/**/coverage.cobertura.xml'
    
    # 発行
    - task: DotNetCoreCLI@2
      displayName: 'Publish Application'
      inputs:
        command: 'publish'
        publishWebProjects: false
        projects: 'backend/backend.csproj'
        arguments: '--configuration $(buildConfiguration) --output $(Build.ArtifactStagingDirectory)'
        zipAfterPublish: true
    
    # アーティファクトの発行
    - task: PublishBuildArtifacts@1
      displayName: 'Publish Build Artifacts'
      inputs:
        PathtoPublish: '$(Build.ArtifactStagingDirectory)'
        ArtifactName: 'drop'

# ステージ 2: セキュリティスキャン
- stage: Security
  displayName: 'Security Scan'
  dependsOn: Build
  condition: succeeded()
  jobs:
  - job: SecurityScan
    pool:
      vmImage: 'ubuntu-latest'
    steps:
    - script: |
        echo "Running security vulnerability scan..."
        # 実際には OWASP Dependency Check などを実行
        dotnet list backend/backend.csproj package --vulnerable
      displayName: 'Vulnerability Scan'

# ステージ 3: デプロイ（Dev環境）
# 注意: デプロイ前に az400-vm が起動していることを確認してください
# az vm start --resource-group rg-az400-handson3 --name az400-vm
- stage: DeployDev
  displayName: 'Deploy to Dev'
  dependsOn: Security
  condition: and(succeeded(), eq(variables['Build.SourceBranch'], 'refs/heads/main'))
  jobs:
  - deployment: DeployDevJob
    displayName: 'Deploy to Dev Environment'
    pool:
      vmImage: 'ubuntu-latest'
    environment: 'Development'
    strategy:
      runOnce:
        deploy:
          steps:
          - task: DownloadBuildArtifacts@1
            inputs:
              buildType: 'current'
              downloadType: 'single'
              artifactName: 'drop'
              downloadPath: '$(System.ArtifactsDirectory)'
          
          # アーティファクトを解凍
          - task: ExtractFiles@1
            displayName: 'Extract Published Files'
            inputs:
              archiveFilePatterns: '$(System.ArtifactsDirectory)/drop/*.zip'
              destinationFolder: '$(System.ArtifactsDirectory)/extracted'
          
          # SSH経由でVMにファイルをコピー
          - task: CopyFilesOverSSH@0
            displayName: 'Copy Files to VM'
            inputs:
              sshEndpoint: 'az400-vm-ssh'
              sourceFolder: '$(System.ArtifactsDirectory)/extracted'
              contents: '**'
              targetFolder: '/var/www/backend'
              cleanTargetFolder: true
              overwrite: true
          
          # VMでアプリケーションを再起動
          - task: SSH@0
            displayName: 'Restart Application on VM'
            inputs:
              sshEndpoint: 'az400-vm-ssh'
              runOptions: 'inline'
              inline: |
                # アプリケーションを停止（既存のプロセスがあれば）
                sudo pkill -f "dotnet.*backend.dll" || true
                
                # systemd サービスファイルを作成（初回のみ）
                if [ ! -f /etc/systemd/system/backend.service ]; then
                  sudo tee /etc/systemd/system/backend.service > /dev/null <<EOF
                [Unit]
                Description=Backend API Service
                After=network.target

                [Service]
                Type=notify
                WorkingDirectory=/var/www/backend
                ExecStart=/usr/bin/dotnet /var/www/backend/backend.dll
                Restart=always
                RestartSec=10
                User=azureuser
                Environment=ASPNETCORE_ENVIRONMENT=Production
                Environment=ASPNETCORE_URLS=http://0.0.0.0:5000

                [Install]
                WantedBy=multi-user.target
                EOF
                
                  sudo systemctl daemon-reload
                  sudo systemctl enable backend.service
                fi
                
                # Application Insights Connection String を設定
                sudo mkdir -p /etc/backend
                echo "APPLICATIONINSIGHTS_CONNECTION_STRING=$(az monitor app-insights component show --app az400-appinsights --resource-group rg-az400-handson3 --query connectionString -o tsv)" | sudo tee /etc/backend/appsettings.json
                
                # サービスを起動
                sudo systemctl restart backend.service
                
                # ステータス確認
                sleep 3
                sudo systemctl status backend.service --no-pager
              readyTimeout: '20000'
```

**VM上のアプリケーションに Application Insights を設定**

> **🎯 AZ-400試験のポイント**: シークレット管理のベストプラクティスは **Azure Key Vault + Managed Identity** です。

---

**方法1: Azure Key Vault + Managed Identity（推奨）**

この方法は**AZ-400試験で推奨**されるベストプラクティスです：
- ✅ シークレットをコード/設定ファイルにハードコードしない
- ✅ Managed Identityによる安全なアクセス制御
- ✅ シークレットのローテーションが容易
- ✅ 監査ログが取得可能

```powershell
# 前提: Key Vault が存在しない場合は作成
$keyVaultName = "kv-az400-handson3"  # グローバルに一意な名前に変更してください

# Key Vault を作成（まだない場合）
az keyvault create `
  --name $keyVaultName `
  --resource-group rg-az400-handson3 `
  --location japaneast `
  --enable-rbac-authorization false

# VMにManaged Identityを有効化（System-assigned）
az vm identity assign `
  --name az400-vm `
  --resource-group rg-az400-handson3

# Managed Identity の Object ID を取得
$vmIdentityObjectId = az vm show `
  --name az400-vm `
  --resource-group rg-az400-handson3 `
  --query "identity.principalId" -o tsv

Write-Host "VM Managed Identity Object ID: $vmIdentityObjectId"

# Application Insights Connection String を Key Vault に保存
az keyvault secret set `
  --vault-name $keyVaultName `
  --name "AppInsights-ConnectionString" `
  --value $connectionString

# VMのManaged IdentityにKey Vaultへのアクセス権限を付与
az keyvault set-policy `
  --name $keyVaultName `
  --object-id $vmIdentityObjectId `
  --secret-permissions get list

# VM内でKey Vaultからシークレットを取得するようにアプリケーションを設定
# SSH経由でVMに接続
ssh azureuser@20.89.144.177

# Azure CLI をVMにインストール（まだない場合）
curl -sL https://aka.ms/InstallAzureCLIDeb | sudo bash

# Managed Identityでログイン
az login --identity

# Key Vault URIを環境変数として設定
sudo mkdir -p /etc/systemd/system/backend.service.d
sudo tee /etc/systemd/system/backend.service.d/override.conf > /dev/null <<EOF
[Service]
Environment="KeyVaultName=$keyVaultName"
EOF

# サービスを再起動
sudo systemctl daemon-reload
sudo systemctl restart backend.service

# SSH接続を終了
exit
```

> **注意**: この方法を使用する場合、アプリケーションコード（`Program.cs`）でKey Vaultからシークレットを読み取る実装が必要です。詳細は Exercise 5.1-補足 を参照してください。

---

**方法2: 環境変数（systemd service）— 学習用の簡易方法**

本番環境では**非推奨**ですが、学習目的で環境変数を使用する方法：

```powershell
# SSH経由でVMに接続
ssh azureuser@20.89.144.177

# systemd サービスの環境変数を設定

[Service]

# サービスを再起動

# ステータス確認

# SSH接続を終了
exit
```

⚠️ **警告**: この方法ではConnection StringがVM上に平文で保存されます。

**5.1-補足: Key Vault統合のためのコード変更（方法1を使用する場合）**

`backend/Program.cs` に以下を追加：

```csharp
using Azure.Identity;
using Azure.Security.KeyVault.Secrets;

var builder = WebApplication.CreateBuilder(args);

// Key Vault から Connection String を取得
var keyVaultName = Environment.GetEnvironmentVariable("KeyVaultName");
if (!string.IsNullOrEmpty(keyVaultName))
{
    var keyVaultUri = new Uri($"https://{keyVaultName}.vault.azure.net/");
    var client = new SecretClient(keyVaultUri, new DefaultAzureCredential());
    
    try
    {
        KeyVaultSecret secret = client.GetSecret("AppInsights-ConnectionString");
        Environment.SetEnvironmentVariable("APPLICATIONINSIGHTS_CONNECTION_STRING", secret.Value);
    }
    catch (Exception ex)
    {
        Console.WriteLine($"Warning: Failed to retrieve secret from Key Vault: {ex.Message}");
    }
}

// 既存のコード...
builder.Services.AddApplicationInsightsTelemetry();
```

dotnet add backend/backend.csproj package Azure.Security.KeyVault.Secrets
